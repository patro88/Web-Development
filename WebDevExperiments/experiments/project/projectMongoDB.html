Data Storage - MongoDB


A lot of data is handled in the project, for backend storage purpose I am using MongoDB which is a no SQL database. In mongoDB there are collections instead of tables and documents (JSON Objects) instead of tupples.
The Coolections for my project are :

Collection Purpose
user This is the collection which stores data about the user i.e. the data to that user enters at time of registration. Also it stores the array of pages , which contains all the widgets created by the user.
question The puspose of this collection is to store all the questions in the database. It has other important properties like answers array, likes array, created by , and tags array.
tag This collection is used to store all the tags created while creating a question. The Question and tags collections have a many to many relation.
